# DRLX Rule Annotations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement DRLX rule-level annotations `@Salience(int)` and `@Description(String)` by extending the canonical ANTLR → IR → proto → runtime pipeline end-to-end, with strict import-based resolution and fail-loud handling of unknowns, duplicates, and non-literal arguments.

**Architecture:** Grammar adds `annotation*` prefix to `ruleDeclaration` (uses the `annotation` rule already inherited from `JavaParser.g4`). The canonical visitor resolves annotation names against compilation-unit imports into a closed `Kind` enum (`SALIENCE`, `DESCRIPTION`), validates literal shape, and rejects everything else with line:col diagnostics. The runtime builder maps `Kind.SALIENCE` to `RuleImpl.setSalience(new SalienceInteger(value))` and `Kind.DESCRIPTION` to `RuleImpl.addMetaAttribute("Description", value)` — both APIs already used by Drools.

**Tech Stack:** Java 17 / records, ANTLR4 (auto-regenerated by Maven), Protocol Buffers (manually regenerated with bundled `protoc-25.5` and checked in), JUnit 5, AssertJ. Spec: `specs/2026-04-20-rule-annotations-design.md`. Active issue: **#3** on `tkobayas/drlx-parser`.

```java
    private List<RuleAnnotationIR> buildRuleAnnotations(DrlxParser.RuleDeclarationContext ctx,
                                                        Map<String, String> annotationImports) {
        if (ctx.annotation() == null || ctx.annotation().isEmpty()) {
            return List.of();
        }

        List<RuleAnnotationIR> annotations = new ArrayList<>();
        java.util.EnumSet<Kind> seen = java.util.EnumSet.noneOf(Kind.class);

        for (DrlxParser.AnnotationContext annCtx : ctx.annotation()) {
            int line = annCtx.getStart().getLine();
            int col = annCtx.getStart().getCharPositionInLine();

            String nameText = annotationNameText(annCtx);
            Kind kind = resolveKind(nameText, annotationImports, line, col);

            if (!seen.add(kind)) {
                throw new RuntimeException(
                        "duplicate @" + kind.name().charAt(0)
                        + kind.name().substring(1).toLowerCase()
                        + " at " + line + ":" + col);
            }

            String rawValue = extractAnnotationLiteral(annCtx, kind, line, col);
            annotations.add(new RuleAnnotationIR(kind, rawValue));
        }

        return List.copyOf(annotations);
    }

    private static String annotationNameText(DrlxParser.AnnotationContext annCtx) {
        if (annCtx.qualifiedName() != null) {
            return annCtx.qualifiedName().getText();
        }
        if (annCtx.altAnnotationQualifiedName() != null) {
            return annCtx.altAnnotationQualifiedName().getText();
        }
        throw new RuntimeException("Malformed annotation at "
                + annCtx.getStart().getLine() + ":"
                + annCtx.getStart().getCharPositionInLine());
    }

    private static Kind resolveKind(String nameText,
                                    Map<String, String> annotationImports,
                                    int line, int col) {
        if (nameText.contains(".")) {
            Kind kind = SUPPORTED_ANNOTATION_KINDS.get(nameText);
            if (kind != null) {
                return kind;
            }
            throw new RuntimeException(
                    "unsupported DRLX rule annotation '@" + nameText + "' at "
                    + line + ":" + col + " — supported: @Salience, @Description");
        }
        String fqn = annotationImports.get(nameText);
        if (fqn != null) {
            return SUPPORTED_ANNOTATION_KINDS.get(fqn);
        }
        throw new RuntimeException(
                "unresolved annotation '@" + nameText + "' at "
                + line + ":" + col + " — missing import?");
    }

    private static String extractAnnotationLiteral(DrlxParser.AnnotationContext annCtx,
                                                   Kind kind, int line, int col) {
        if (annCtx.elementValue() == null) {
            throw new RuntimeException("@" + kind + " expects one argument at " + line + ":" + col);
        }
        String text = annCtx.elementValue().getText();
        switch (kind) {
            case SALIENCE -> {
                try {
                    return String.valueOf(Integer.parseInt(text));
                } catch (NumberFormatException e) {
                    throw new RuntimeException(
                            "@Salience expects int literal, got '" + text + "' at " + line + ":" + col);
                }
            }
            case DESCRIPTION -> {
                if (text.length() >= 2 && text.startsWith("\"") && text.endsWith("\"")) {
                    return text.substring(1, text.length() - 1);
                }
                throw new RuntimeException(
                        "@Description expects string literal, got '" + text + "' at " + line + ":" + col);
            }
            default -> throw new IllegalStateException("Unhandled annotation kind: " + kind);
        }
    }
```

**Notes on the implementation:**
- `Integer.parseInt(text)` rejects `"ten"` (quotes in the text), `p.age + 10` (not numeric), `10L` (trailing L), `0xA` (hex) — all "non int literal" forms fail uniformly with `NumberFormatException`. It correctly accepts `"10"`, `"-5"`, `"+7"`. This is the cleanest validator for Q3 (literal int only).
- The `elementValuePairs` case (named form `@Salience(value = 10)`) falls into `annCtx.elementValue() == null` branch above — we reject the named form as "expects one argument". That's acceptable: DRLXXXX.md only shows positional-argument form. If we later want to support named, we can extend `extractAnnotationLiteral` to also handle `elementValuePairs`.
- `EnumSet.add(kind)` returns false on duplicate — clean duplicate detection.

- Create: `drlx-parser-core/src/test/java/org/drools/drlx/builder/syntax/RuleAnnotationsTest.java`

**Strategy:** TDD — write the first failing test. Without runtime wiring (Task 7), salience is extracted into IR but never applied to `RuleImpl.setSalience(...)`. Both rules will therefore have the default zero salience and fire in agenda-insertion order (which is implementation-defined but in practice is insertion-reverse or insertion-forward). We capture firing order via a `DefaultAgendaEventListener` and assert the observed sequence matches `@Salience` priority. Without runtime wiring, that assertion will fail.

**Why a listener, not a `global`:** DRLX has no `global` declaration in the grammar yet — the only way to observe firing from outside is a `KieSession` event listener.

Create `drlx-parser-core/src/test/java/org/drools/drlx/builder/syntax/RuleAnnotationsTest.java`:

```java

import java.util.ArrayList;
import java.util.List;

import org.drools.drlx.builder.DrlxRuleBuilder;
import org.drools.drlx.domain.Person;
import org.junit.jupiter.api.Test;
import org.kie.api.KieBase;
import org.kie.api.definition.rule.Rule;
import org.kie.api.event.rule.AfterMatchFiredEvent;
import org.kie.api.event.rule.DefaultAgendaEventListener;
import org.kie.api.runtime.KieSession;
import org.kie.api.runtime.rule.EntryPoint;

import static org.assertj.core.api.Assertions.assertThat;
import static org.assertj.core.api.Assertions.assertThatThrownBy;

class RuleAnnotationsTest extends DrlxBuilderTestSupport {

    @Test
    void testSalienceAffectsFiringOrder() {
        // HighSalience (10) must fire before LowSalience (5) for the same fact.
        // An AgendaEventListener captures firing order externally.
        final String rule = """
                package org.drools.drlx.parser;

                import org.drools.drlx.domain.Person;
                import org.drools.drlx.annotations.Salience;

                unit MyUnit;

                @Salience(5)
                rule LowSalience {
                    Person p : /persons[ age > 18 ],
                    do { System.out.println("low"); }
                }

                @Salience(10)
                rule HighSalience {
                    Person p : /persons[ age > 18 ],
                    do { System.out.println("high"); }
                }
                """;

        withSession(rule, kieSession -> {
            final List<String> fired = new ArrayList<>();
            kieSession.addEventListener(new DefaultAgendaEventListener() {
                @Override
                public void afterMatchFired(AfterMatchFiredEvent event) {
                    fired.add(event.getMatch().getRule().getName());
                }
            });

            final EntryPoint entryPoint = kieSession.getEntryPoint("persons");
            entryPoint.insert(new Person("Alice", 30));

            final int firedCount = kieSession.fireAllRules();

            assertThat(firedCount).isEqualTo(2);
            assertThat(fired).containsExactly("HighSalience", "LowSalience");
        });
    }
}
```
